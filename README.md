# RYZE · Monthly Goal Dashboard

Live, auto-refreshing web dashboard for the copywriting team's monthly output. Data is generated by the `monthly-goal-tracker` skill and committed daily.

## How it works

```
track_goals.py --emit-json  →  dashboard.json  →  index.html fetches + renders
                    ↓
             git commit + push                       (browser polls every 60s)
```

## View it

Once GitHub Pages is enabled on this repo:

- https://pauliusryze.github.io/livemonthlygoaldashboard/

Or the raw JSON directly:
- https://raw.githubusercontent.com/pauliusryze/livemonthlygoaldashboard/main/dashboard.json

## Setup (one-time)

1. Enable GitHub Pages in this repo's Settings → Pages → Source: `main` branch, root (`/`) folder.
2. Wait ~1 min. Visit the URL above.

## Update cadence

The `com.ryze.goal-tracker.plist` launchd agent on Paul's Mac runs `track_goals.py --emit-json` at 23:30 local time and pushes `dashboard.json` to this repo. The HTML auto-refreshes every 60 seconds, so the page stays current while you keep the tab open.

## Files

| File | What |
|---|---|
| `index.html` | Self-contained dashboard (no build step) |
| `dashboard.json` | Snapshot of the current month's goal state, updated nightly |
| `README.md` | This file |

## Manual refresh

```bash
cd /Users/paulius/Downloads/copyclaw/skills/monthly-goal-tracker/scripts
python3 track_goals.py --emit-json /Users/paulius/Downloads/livemonthlygoaldashboard/dashboard.json
cd /Users/paulius/Downloads/livemonthlygoaldashboard
git add dashboard.json && git commit -m "refresh $(date +%F)" && git push
```
